域名解析模块tld和tldextract分析

  • python
  • tld
  • tldextract
  • 域名解析
域名域名解析
  • tld模块链接
  • tldextract模块链接
  • tld和tldextract比较

    python对于网络操作特别的方便，其中尤其是对域名的分解操作，这次主要介绍两个库tld和tldextract,python也有标准解析库urlparse,在这里不多介绍，有兴趣的同学可以参考python的标准库，有详细的分析与介绍。

tld模块链接

代码如下：

from tld import get_tld  
res = get_tld("http://www.baidu.com/,as_object=True)
print res            # www.baidu.com
print res.subdomain  # www
print res.domain     # baidu
print res.fuffix     # com
print res.tld        # baidu.com

tldextract模块链接

代码如下：

import tldextract
ext = tldextract.extract('http://forums.bbc.co.uk')
print ext.subdomain,ext.domain,ext.suffix

注意:，如果直接使用tldextract.extract,则程序运行第一次的时候，库会更新域名后缀列表，根据网络情况会很慢，可以取消更新，直接使用提取功能，详见网址,英语较差的同学直接测试代码即可。

tld和tldextract比较

    强烈建议使用tldextract，因为根据IANA机构发布最新消息，顶级域名后缀已经达到1000多个，域名的开放，使更多的后缀在更新中，而tld从前几年开始就已经不在更新，其后缀域名库已经远远达不到要求，因为项目需要，自己一直修改其标准库，每次都要更新，特别项目部署的时候，更是麻烦。
    同时，tldextract实时更新，目前自己遇到的后缀基本都有，若没有只是稍微更新即可，所以强烈建议使用tldextract。
