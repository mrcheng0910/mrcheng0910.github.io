MySQL数据库学习记录--DML语句

  • dml
  • linux
  • mysql
  • mysql命令
mysql命令
  • 1. 插入记录
  • 2. 更新记录
  • 3. 删除记录
  • 4. 查询记录

DML操作是对数据库中表记录进行操作，主要包括表记录的插入、更新、删除和查询，是平常使用最多的操作。

1. 插入记录

表建好后，就可以插入记录

语法：insert into tablename (field1,field2,...) values(value1,value2,...),也可以不用指定字段，但是顺序应该和字段排序的顺序一致。注意:含可空字段、非空但是有默认值的字段、自增字段，可以不用在insert后的字段列表里出现。另外，insert可以一次性插入多条记录，格式如下： 

insert into tablename(f1,f2,...) values
(r1,r2,...),
(r1,r2,..),
...;

2. 更新记录

表里记录可以使用update命令进行更改，语法如下：

update tablename set field1=value1,filed2=value2,... [where condition]

在mysql中，update可以同时更新多个表中的数据，语法如下：

update t1,t2,..tn set t1.field1=expr1,tn.fieldn=exprn [where condition]

注意: 多表更新的语法更多用在根据一张表的字段，来动态更新另一张表的字段。

3. 删除记录

语法：

delete from tablename [where condition]  //删除一张表中内容
delete from t1,t2,...,tn [where conditon]  //同时删除多个表

注意： 不加where条件时，会删除所有数据，一定小心。

4. 查询记录

基本语法：

select * from tablename [where conditon]
  • 查询不重复记录
    有时候需要将表中记录去掉重复显示出来，可以使用distinct关键字来实现 
select distinct field1 from tablename
  • 条件查询
    即添加where条件进行判断，语法如下：
select filed from tablename where filed = values
  • 排序和限制
    我们经常会有这样的需求，取出按照某个字段进行排序后的记录集，这就使用到了排序操作，用order by来实现，语法如下：
select * from tablename [where conditon] [order by field1 [desc\asc],field2[desc\asc]]

其中desc和asc是排序顺序关键字，desc表示降序，asc为升序，默认为升序，order by后面可以有多个不同排序字段。
对于排序后的记录，如果希望只是显示部分，则使用limit关键字实现，语法如下：

select ... [limit offset_start,row_count]

offset_start表示记录的起始偏移量，row_count表示显示的行数，默认起始变量为0，只需要填写行数即可。

  • 聚合
    很多情况下，用户需要进行汇总操作，这时候就需要使用到sql的聚合操作。语法如下：
select [field1,field2,...,fieldn] fun_name
from tablename [where conditon]
[group by field,field2,...fieldn [with rollup]]
[having where_contition]

参数说明：
fun_name 表示要进行的聚合操作，常用有sum、count、max、min。
group by 表示要进行分类聚合的字段
with rollup 是可选语法，表明是否对分类聚合后的结果进行再汇总。
having 关键字表示对分类后的结果再进行条件过滤。
注意：having和where的区别在于，having是对聚合后的结果进行条件过滤，而where是聚合前就对记录过滤。

  • 表连接
    当需要同时显示多个表中的字段的时候，就需要用表连接来实现这样的功能，从大类上分，表连接分为内连接和外连接，他们之间最大的区别在于，内连接仅选出表中互相匹配的记录，而外连接会选出其他不匹配的记录，最常用的是内连接。内连接基本语法如下： 
select field1,field2 from tbname1,tbname2 where tbname1.field3 = tbname2.field4

外连接又分为左连接和右连接，具体定义如下：
左连接：包含所有左边表中的记录甚至是右边表中没有和它匹配的记录。
右连接：包含所有右边表中的记录甚至是左边表中没有和它匹配的记录。

  • 子查询
    当进行查询的时候，需要的条件是另一个select语句的结果，这个时候就用到子查询，用于子查询的关键字主要包括in、join、=、!=、exists、not exists等.如果子查询记录唯一，使用=代替in。

  • 记录联合
    将两个表的数据按照一定的条件查询出来后，将结果合并到一起显示，这个时候需要使用union和union all关键字来实现该功能，具体语法如下：
select * from t1
union\union all
select * from t2
...
select * from tn

union和union all的区别是union all是把结果直接合并到一起，而union是将结果union all后的结果distinct后的去重结果。
