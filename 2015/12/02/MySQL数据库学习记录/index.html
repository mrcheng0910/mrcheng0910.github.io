MySQL数据库学习记录--DDL语句

  • mysql

从接触mysql开始就一直使用navicat for mysql来进行数据库的操作，确实很方便，但是有一些更高的功能却无法使用，以及对数据库不能深刻理解，所以买了《深入浅出mysql数据库开发、优化与管理维护 第二版》，电子书下载链接，通过记录学习，加深对mysql的理解。同时也是自己的项目需求要达到千万级别的数据的存储以及操作。

1. 登录

语法:mysql -uroot -p,然后输入正确的密码，即可进入到数据库中，-u后面是连接数据库的用户名，默认是root，另外命令的结束符，用;或者\g结束。

2. 创建数据库

语法：create database dbname，dbname为创建的数据库的名称。

3. 查看系统中已存在的数据库

语法:show databases，就会列出所有系统中所有的数据库。

4. 选择操作的数据库

语法:use dbname，dbname为想要操作的数据库名称。

5. 查看某数据库中包含的所有表

语法:show tables，会显示出该数据库中所有的表，该命令操作必须是在确定的数据库中才可以。

6. 删除数据库

语法：drop database dbname，dbname为要删除的数据库名称

7. 创建表

语法： 

create table tablename(
col_name1 col_type1 constraints,
col_name1 col_type1 constraints)

col_name为列的名字，col_type为数据类型，constraints为这个列的约束条件。

8. 查看表的定义

语法：desc tablename,tablename即为所要查看的表的名称。
但是上述显示的信息还不够全面，可以使用以下命令来查看表的更多定义信息。
命令：show create table tbname \g,其中tbname为表名称，\g是用来使得记录能够按照字段竖向排列，以便更好显示内容较长的记录。这个命令可以显示出表的engine(存储引擎)和charset(字符集)等信息。

9. 删除表

语法：drop table tbname，tbname为要删除的表的名称

10. 修改表

备注：first|after column_name是修改字段在表中的位置。

  • 修改表的类型
    语法：alter table tablename modify[column] column_definition[first|after clo_name]

  • 增加表字段
    语法：alter table tablename add[column] column_definition[first|after col_name]

  • 删除表字段
    语法：alter table tablename drop[column] col_name

  • 字段改名
    语法：alter table tablename change[column] old_col_name column_definiton [first|after col_name]
    例如：alter table tbname change age age1 int(4),该例子就是将字段age名称改为age1，且字段类型改为int(4)

  • 修改字段排序顺序
    前面介绍的字段增加和修改方法中，都有一个可选项first|after column_name，修改字段在表中位置。
  • 更改表名
    语法：alter table tablename rename [to] new_tablename
